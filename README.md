Linglow-Pure

Linglow-Pure 是一款专为移动端优化的“小手机”风格 AI 聊天 Web 应用。它是 Linglow 系列的前端开源版本，主打完全本地化存储与极致的拟人化交互体验。

GitHub 家族项目：

Linglow-Pure (本项目): 纯前端版本，数据全本地缓存，隐私安全最高。

Linglow-chat-community: 含后端版本，支持服务器部署与社区功能。

✨ 项目核心优势（解决的痛点）
1. 拟人化：单次多条消息发送
传统的 AI 聊天通常需要用户发一条、AI 回一条，或者多次手动点击“继续生成”。

痛点解决：Linglow 支持用户一次性发送多条消息，AI 会根据语境自动处理并连续回复，无需用户频繁点击，交互感更接近真实的社交软件，充满“活人感”。

2. 直观化：已读/未读 UI 状态
参考了现代社交软件（如 WhatsApp/Telegram）的设计语言。

痛点解决：聊天界面集成了消息状态指示（已读/未读/发送中），让用户对 AI 的响应状态一目了然。

3. 智能化：增量记忆管理 (Memory V3)
项目内置了复杂的增量记忆算法。

痛点解决：

自动关联：每次生成新记忆时，系统会自动匹配并关联之前的旧记忆。

节约 Token：通过精简和总结，在保证记忆连贯性的同时，显著减少发送给 API 的上下文 Token 数量。

整体认知：让 AI 能够随着聊天深入，形成对用户稳定、长期的认知，而不是“转头就忘”。

4. 高级功能：长期记忆碎片化检索 (Beta)
痛点解决：针对长期的海量记忆，Linglow 会筛选出与当前话题最相关的部分进行填充。这既避免了全量加载导致的上下文污染，也有效控制了 API 开销。

🔒 隐私与安全
绝对安全：本项目所有数据（设置、API Key、聊天记录、长期记忆）均通过 localStorage 存储在用户手机本地。

无后端依赖：Pure 版本不设服务器中转，您的隐私只属于您自己。

🚀 核心功能清单
[x] 多角色管理：支持创建、导入多个 AI 角色，每个角色拥有独立的记忆和配置。

[x] 仿原生 UI：丝滑的移动端适配，包含微信/手机桌面风格的交互设计。

[x] 长期记忆总结：可自定义总结阈值（如每 50 条消息自动总结一次）。

[x] 批量消息操作：支持多选、合并转发、收藏及批量删除。

[x] 分支管理：支持导出当前分支的聊天记录为 JSON，方便迁移或备份。

🛠️ 技术栈
前端：原生 HTML5 / CSS3 (Flexbox/Grid) / JavaScript (ES6+)。

存储：浏览器 localStorage（用于大规模记录）。

兼容性：针对移动端内核优化，支持作为 PWA 或添加到桌面使用。

📦 部署与使用
直接运行：克隆仓库后，直接在浏览器打开 index.html 即可开始使用。

配置 API：

点击设置 -> 服务商设置。

输入您的 OpenAI 兼容格式的 API Key 和 Base URL。

开始聊天：创建角色，享受流畅的 AI 交互体验。
